# Паттерн Адаптер (Adapter Pattern)

## Описание

Проект демонстрирует реализацию паттерна проектирования "Адаптер" на языке C#. 
Паттерн Адаптер позволяет объектам с несовместимыми интерфейсами работать вместе.

В данном примере показано, как адаптер преобразует интерфейс `CardInfo` в интерфейс `Message`, чтобы сервис выпуска карт (`CardIssueService`) мог работать с данными карты.

## Структура проекта

### Основные классы

1. **CardInfo** - поля для карты которая будет выпущена
2. **Message** - класс выводящий сообщения инициализированной карты
3. **CardIssueService** - сервис, который работает с сообщениями для выпуска карт
4. **CardIssueServiceAdapter** - адаптер, преобразующий CardInfo в Message
5. **CardManager** - выпуск карты

## Взаимодействие между компонентами

1. **Program** создает экземпляр **CardManager**
2. **CardManager** в методе **Issue** создает **CardIssueService** и **CardIssueServiceAdapter**
3. **CardIssueServiceAdapter** адаптирует интерфейс **CardInfo** к интерфейсу **Message**
4. **CardIssueService** работает с объектами **Message** для выпуска карт
5. Результат возвращается через цепочку вызовов обратно в **Program**

## Подробное описание классов

### CardInfo
Поля для идентификации создаваемой карты:
- `Id` - уникальный идентификатор карты
- `StarDate` - дата начала действия карты
- `ExepirationDate` - дата окончания действия карты
- `message` - сообщение, связанное с картой

### Message
Простой класс сообщения:
- `Content` - содержимое сообщения по выпуску карт

### CardIssueService
Сервис выпуска карт:
- Метод `Issue(Message message)` - выпускает карту с заданным сообщением

### CardIssueServiceAdapter
Адаптер для сервиса выпуска карт:
- Реализует интерфейс **ICardIssueServiceAdapter**
- В конструкторе принимает **CardIssueService**
- Метод `Issue(CardInfo cardInfo)` адаптирует **CardInfo** к **Message** и вызывает метод **Issue** у **CardIssueService**

### CardManager
Менеджер для управления картами:
- Метод `Issue(CardInfo cardInfo)` - выпускает карту с заданными параметрами

### Program
Основной класс программы:
- Создает объект **CardInfo** инициализирует введенными значениями дефолтные значения
- Использует **CardManager** выпускает карты с заданными параметрами в Main

## Применение паттерна Адаптер

Паттерн Адаптер используется в этом примере для решения следующей проблемы:
- У нас есть сервис **CardIssueService**, который работает с объектами **Message**
- У нас есть данные карты в формате **CardInfo**
- Чтобы использовать **CardIssueService** с **CardInfo**, мы создаем адаптер **CardIssueServiceAdapter**, который преобразует **CardInfo** в **Message**

Это позволяет нам использовать существующий сервис без изменения его кода, просто добавив адаптер для преобразования интерфейсов.
